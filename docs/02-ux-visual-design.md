# GymGymGym 信息架构与交互视觉设计

## 1. 设计原则

1. 训练中少打字：优先使用按钮、标签、数字步进。
2. 记录密度高：一屏尽量看到动作、重量、次数、完成状态。
3. 状态明确：当前训练是否已保存、是否进行中要明显。
4. 私人产品：视觉温暖、克制，不做复杂社交化包装。
5. 中英兼容：动作名可英文展示，中文别名优先。

## 4. 视觉规范

### 4.1 色彩

| Token | 用途 | 值 |
|---|---|---|
| `--color-bg` | 页面背景 | `#f7f5f0` |
| `--color-card` | 卡片背景 | `#ffffff` |
| `--color-primary` | 主按钮/强调 | `#111827` |
| `--color-accent` | 数据强调 | `#ef7d32` |
| `--color-muted` | 次级文字 | `#6b7280` |
| `--color-border` | 边框 | `#e5e7eb` |

### 4.2 字体层级

| 场景 | 字号 | 字重 |
|---|---|---|
| 页面标题 | 44rpx | 700 |
| 区块标题 | 34rpx | 700 |
| 正文 | 28rpx | 400 |
| 辅助说明 | 24rpx | 400 |
| 数据数字 | 40rpx | 700 |

### 4.3 组件

- 卡片：圆角 28rpx，内边距 28rpx，轻微阴影。
- Chip：圆角胶囊，多选状态使用深色背景。
- 主按钮：黑底白字。
- 次按钮：浅灰背景深色字。
- 危险操作：仅使用文字按钮，不突出。

## 5. 关键交互

### 5.1 添加普通动作

1. 点击 `+ 普通动作`。
2. 打开动作选择器。
3. 优先在“常用收藏”区选择动作，也可以搜索全部动作。
4. 选择动作。
5. 创建 `single` block。
6. 自动生成第一组空 set。

### 5.2 添加超级组

1. 点击 `+ 超级组`。
2. 优先在“常用收藏”区选择动作，也可以搜索全部动作。
3. 选择 2 个动作。
4. 创建 `superset` block。
5. 自动生成 round 1 的两个 set。

### 5.3 保存训练

1. 点击保存。
2. 校验至少有一个有效 set。
3. 写入 `workout_sessions`、`workout_blocks`、`workout_sets`。
4. 调用统计重算云函数或本地提示用户稍后刷新。
5. 返回完成状态。

### 5.4 复制同类训练

1. 训练页最近训练默认展示最近 5 条。
2. 用户可切换 `全部 / 同目标 / 同场所 / 同目标+场所`。
3. 筛选依据当前训练场景里的目标和场所实时变化。
4. 点击“复制”后带入历史训练块、动作、重量、次数、RPE 和热身/力竭标记。
5. 保存时创建新的训练记录，不修改原历史记录。
