# V0.2 聚焦版重构说明

## 产品原则

V0.2 只围绕三个动作：生成一套可执行的训练、在训练中记录实际重量/次数、训练后看近期摘要与 PR。减少让用户填写表单、维护目标和管理数据的负担。

## 页面范围

1. 训练
   - 打开页面自动生成推荐计划。
   - 用户只输入一句自然语言，例如“今天练胸肩，只有哑铃，状态好，45 分钟”。
   - 系统识别部位、器材、状态、时间和突破/恢复意图。
   - 基于历史动作统计给出参考重量、组数、次数和 RPE。
   - 可选择过去日期，直接补录历史训练。
   - 开始后生成区域收缩，页面聚焦实际组数、重量和次数。
2. 动作库
   - 分页浏览全部 1324 个开源动作。
   - 搜索、部位和器材筛选。
   - 显示动作示范图和 attribution。
   - 1324 张图片部署于 CloudBase 静态托管 `exercise-media/images/`。
3. 复盘
   - 只显示近 7/30 天训练摘要。
   - 只显示 PR Top 3。
4. 我的
   - 昵称、训练经验、主要目标、常用场地、常用器材和一句话习惯。

## 删除/暂停的功能

- 结构化具体目标管理。
- JSON 数据导出入口。
- 档案顶部累计次数/组数/容量。
- 完整动作表现长列表。
- 训练前的大表单和手动逐动作搭计划。
- 收藏、超级组等非核心入口暂不放在主路径。

## 智能计划策略

当前环境未加入 CloudBase 小程序 AI 成长计划，旧免费 Token 包已过期。因此 V0.2 先采用可解释的自然语言规则 + 历史数据推荐，不让核心训练生成依赖未开通的模型资质：

- 自然语言解析部位、器材、疲劳、突破和训练时长。
- 动作从完整动作库筛选。
- 有历史时参考最近重量/次数；突破时小幅加重，疲劳时减重 15%。
- 无历史时重量留空，默认 3 组 × 10 次、RPE 7，避免编造不安全重量。

后续加入成长计划或 Token Credits 后，可把同一输入和历史数据交给 AI 模型，当前规则作为稳定 fallback。
