# GymGymGym 产品需求定义

## 1. 背景

GymGymGym 是一个私人健身记录微信小程序，最初只服务 2 个用户：本人和伴侣。产品目标不是做公开动作百科，而是把每次训练沉淀为长期可查看、可复盘、可辅助决策的个人训练数据资产。

基础动作库使用 `hasaneyldrm/exercises-dataset`。该库提供 1,324 个动作、动作分类、器械、目标肌群、多语言说明和媒体路径。本项目在其上增加中文别名、常用动作、自定义动作、个人训练记录和个人统计。

## 2. 目标用户

| 用户 | 特征 | 主要诉求 |
|---|---|---|
| 用户 A | 有持续训练习惯 | 快速记录每次训练，知道某动作历史最好成绩 |
| 用户 B | 训练经验可能不同 | 简单记录，查看常练动作和进步情况 |

## 3. 核心目标

1. 训练中记录足够快，不干扰训练节奏。
2. 能表达真实训练结构：普通组、超级组、后续可扩展循环组/递减组。
3. 自动汇总个人动作档案，减少手工翻历史记录。
4. 用当天目标、场地、器械、状态辅助安排训练。
5. 数据归属清晰，每个人只看/改自己的训练数据。

## 4. MVP 范围

### 4.1 今日训练

- 创建一次训练 session。
- 记录训练标题、日期、场地、训练目标、训练模式。
- 记录当天可用器械、想练部位、能量状态、时间限制。
- 根据当天想练部位、可用器材、身体状态、训练策略和收藏动作生成轻量推荐动作。
- 添加训练块 block：
  - `single`：普通单动作。
  - `superset`：超级组，创建时可选择 2、3 或 4 个动作，并按 round 记录。
- 为每个动作记录 set：
  - 重量 kg。
  - 次数 reps。
  - RPE。
  - 是否热身。
  - 是否力竭。
  - 备注。
- 保存并结束训练。

### 4.2 动作库

- 使用内置小样本数据启动开发。
- 支持后续导入完整 `exercises-dataset`。
- 支持搜索动作名称、中文名、目标肌肉。
- 支持按身体部位和器械筛选。
- 支持查看动作详情和中文说明。
- 支持收藏/取消收藏常用动作。
- 训练动作选择器优先展示收藏动作，搜索结果中收藏动作置顶。
- 支持从动作详情加入今日训练。

### 4.3 个人档案

- 汇总训练总次数。
- 汇总总组数、总次数、总容量。
- 展示近 7 天和近 30 天训练摘要：训练次数、有效组数、总次数、训练容量。
- 展示 PR 关注 Top 3：按估算 1RM 或最大重量挑出当前强项。
- 支持导出当前用户的训练、训练块、组明细、动作统计和具体目标为 JSON 备份。
- 按动作展示：
  - 总组数。
  - 总次数。
  - 最大重量。
  - 最大容量。
  - 估算 1RM。
  - 最近训练时间。

### 4.4 目标/设置

- 设置昵称。
- 设置长期目标。
- 设置训练水平。
- 设置默认场地。
- 设置常用器械。
- 保存常用/收藏动作列表。
- 创建多个结构化具体目标：目标名称、目标类型、目标指标、关联动作、重点部位、截止日期和说明。
- 支持将具体目标标记为完成、恢复进行中或删除。

## 5. 非 MVP 范围

以下能力后续再做：

- AI 自动生成完整训练计划。
- 图表趋势页。
- Apple Health / Keep / Garmin 数据同步。
- 社交分享。
- 多用户团队管理。
- 完整商业化授权媒体分发。

## 6. 关键业务规则

### 6.1 训练结构

一次训练由多个 block 组成。block 是训练结构的最小组织单位。

```text
session
  block 1: single 卧推
  block 2: superset 哑铃弯举 + 侧平举
  block 3: single 绳索下压
```

### 6.2 超级组

超级组不是多个普通动作简单相邻，而是一个 block 内按 round 组织：

```text
round 1:
  动作 A set
  动作 B set
round 2:
  动作 A set
  动作 B set
```

### 6.3 估算 1RM

MVP 使用 Epley 公式：

```text
estimated_1rm = weight_kg * (1 + reps / 30)
```

热身组不参与 PR 和 1RM 计算。

### 6.4 权限

- 微信小程序端使用 CloudBase 自动身份。
- 训练数据按 `_openid` 或 `user_openid` 归属。
- 管理型统计云函数通过 `OPENID` 限制只计算当前调用者数据。

## 7. 验收标准

1. 小程序可在微信开发者工具打开。
2. `app.js` 已配置 CloudBase 环境。
3. 可创建训练并保存到 CloudBase 数据库。
4. 可添加普通组和超级组。
5. 可查看动作库并搜索筛选。
6. 可查看个人统计。
7. 可保存个人目标设置。
8. 可在动作库收藏常用动作，并在训练动作选择器中优先选择。
9. 可根据当天场景看到推荐动作并一键加入训练。
10. 个人档案可查看近期训练摘要和 PR 关注。
11. 可创建并管理多个具体健身目标。
12. 可导出当前用户个人数据 JSON 备份。
13. 云函数可部署。
14. 代码和文档已推送 GitHub。
