# GymGymGym 手动验收流程

## 1. 微信开发者工具基础检查

1. 用微信开发者工具打开仓库根目录。
2. 确认 `project.config.json` 里是你的真实小程序 AppID。
3. 确认云开发环境为：

```text
code-realtime-d7gbuxrbze297e600
```

4. 编译小程序，确认首页进入“训练”页，无白屏。

## 2. 目标设置验收

1. 打开“目标”。
2. 填写昵称。
3. 选择训练水平、默认目标、默认场地。
4. 选择 2-3 个常用器材。
5. 填写长期目标描述。
6. 点击“保存设置”。
7. 退出页面再进入，确认内容仍然存在。

通过标准：用户资料保存到 `users` 集合，且不会覆盖 `created_at`、`role` 等已有字段。

## 3. 动作库验收

1. 打开“动作库”。
2. 搜索“卧推”或 “bench”。
3. 选择部位筛选，例如“胸部”。
4. 选择器械筛选，例如“杠铃”。
5. 点击动作卡片查看说明。
6. 点击“收藏”，确认按钮变为“已收藏”。
7. 点击“只看收藏”，确认只显示已收藏动作。
8. 退出动作库再进入，确认收藏仍然存在。

通过标准：已导入完整动作库时可看到云端数据；未导入时至少能看到内置 seed 动作。

## 4. 今日训练验收

1. 打开“训练”。
2. 修改训练标题，例如“胸肩三头”。
3. 选择场所、目标、今天策略、身体状态。
4. 勾选今天想练的部位和可用器材。
5. 检查“根据今天场景推荐”是否出现匹配动作，点击其中一个动作的“加入”。
6. 点击“开始训练”。
7. 点击“普通组”，选择 1 个动作。
8. 填写重量、次数、RPE。
9. 点击“加一轮”，再填一组。
10. 点击“超级组”，连续选择 2 个动作。
11. 填写每个动作的重量、次数、RPE。
12. 标记某一组为热身或力竭。
13. 点击“保存并结束训练”。

通过标准：

- `workout_sessions` 新增一条 completed 记录。
- `workout_blocks` 有普通组和超级组记录。
- `workout_sets` 有每一组动作明细。
- 如果动作已收藏，动作选择器顶部出现“常用收藏”，收藏动作也会在搜索结果中置顶。
- 推荐动作会随想练部位、可用器材、策略和身体状态变化；点击“加入”会创建普通训练块。
- 保存后最近训练中能看到刚保存的训练。

## 5. 训练详情验收

1. 在“训练”页点击最近训练。
2. 确认进入训练详情。
3. 确认标题、日期、场所、目标显示正确。
4. 确认普通组、超级组分块展示。
5. 确认每组的动作名、轮次、重量、次数正确。
6. 修改训练信息中的标题、日期、场所、目标和备注，点击“保存训练信息”。
7. 退出再进入详情，确认训练信息修改被保留。
8. 修改某一组的重量、次数或 RPE，点击“保存”。
9. 返回“档案”页确认统计已重算，最大重量或容量随修改更新。
10. 回到训练详情，删除一组误录数据。
11. 删除一个测试训练块，确认页面刷新后该块消失。

通过标准：编辑和删除操作只影响当前用户自己的训练记录；保存/删除后会调用 `recalculateStats`，档案页不再保留被删除训练组造成的旧统计。

## 5.1 复制上次训练验收

1. 在“训练”页找到“最近训练”。
2. 先切换“同目标”“同场所”“同目标+场所”筛选，确认列表数量和展示内容会随当前目标/场所变化。
3. 点击某条历史训练右侧“复制”。
4. 确认训练块、动作、重量、次数、RPE 被带入当前训练编辑区。
5. 修改标题、日期或某一组重量。
6. 点击“保存并结束训练”。
7. 打开最近训练中的新记录，确认它是新的训练记录，旧记录没有被修改。

## 6. 个人档案验收

1. 打开“档案”。
2. 点击“重算”。
3. 确认训练次数、有效组数、总次数、总容量更新。
4. 确认动作表现里显示最大重量和估算 1RM。

通过标准：

- `recalculateStats` 云函数执行成功。
- `exercise_stats` 按当前 OpenID 写入统计。
- 如果云函数暂不可用，页面仍可基于最近 `workout_sets` 做轻量 fallback 展示。

## 7. 真机验收

1. 微信开发者工具生成预览码。
2. 两个人分别扫码体验。
3. 两个人各保存一次训练。
4. 确认两个人只能看到自己的训练和档案。

## 8. 回归检查命令

每次修改后至少运行：

```bash
npm run check
git diff --check
```
