# GymGymGym 手动验收流程

## 1. 微信开发者工具基础检查

1. 用微信开发者工具打开仓库根目录。
2. 确认 `project.config.json` 里是你的真实小程序 AppID。
3. 确认云开发环境为：

```text
code-realtime-d7gbuxrbze297e600
```

4. 编译小程序，确认首页进入“训练”页，无白屏。

## 2. 目标设置验收

1. 打开“目标”。
2. 填写昵称。
3. 选择训练水平、默认目标、默认场地。
4. 选择 2-3 个常用器材。
5. 填写长期目标描述。
6. 点击“保存设置”。
7. 退出页面再进入，确认内容仍然存在。
8. 在“创建具体目标”填写“卧推达到 100kg”，选择力量突破、目标重量 100kg、关联动作和截止日期。
9. 点击“创建目标”，确认目标出现在“进行中的目标”。
10. 点击“标记完成”，确认目标进入“已完成”；再点击“恢复进行中”，确认可恢复。
11. 创建一条测试目标并删除，确认列表刷新且目标不再出现。

通过标准：用户资料保存到 `users` 集合，且不会覆盖 `created_at`、`role` 等已有字段；具体目标保存到 `user_goals`，只显示当前用户自己的目标。

## 3. 动作库验收

1. 打开“动作库”。
2. 搜索“卧推”或 “bench”。
3. 选择部位筛选，例如“胸部”。
4. 选择器械筛选，例如“杠铃”。
5. 点击动作卡片查看说明。
6. 点击“收藏”，确认按钮变为“已收藏”。
7. 点击“只看收藏”，确认只显示已收藏动作。
8. 退出动作库再进入，确认收藏仍然存在。

通过标准：已导入完整动作库时可看到云端数据；未导入时至少能看到内置 seed 动作。

## 4. 今日训练验收

1. 打开“训练”。
2. 修改训练标题，例如“胸肩三头”。
3. 选择场所、目标、今天策略、身体状态。
4. 勾选今天想练的部位和可用器材。
5. 检查“根据今天场景推荐”是否出现匹配动作，点击其中一个动作的“加入”。
6. 检查“部位训练间隔”是否显示历史部位；今天或昨天刚练过的部位应使用警示色，间隔较久的部位显示天数。
7. 点击“开始训练”。
8. 点击“普通组”，选择 1 个动作。
9. 填写重量、次数、RPE。
10. 点击“加一轮”，再填一组。
11. 将超级组动作数量切换为 3，点击“添加超级组”，连续选择 3 个不同动作；再尝试重复选择同一动作，确认被阻止。
12. 填写每个动作的重量、次数、RPE，并点击“加一轮”确认新一轮仍包含全部 3 个动作。
13. 标记某一组为热身或力竭。
14. 点击“保存并结束训练”。
15. 新建一个只有空训练组的训练，确认“保存并结束训练”提示至少填写重量或次数。
16. 只点击“开始训练”但不完成，重新进入后确认该草稿不出现在最近训练或档案统计中。

通过标准：

- `workout_sessions` 新增一条 completed 记录。
- `workout_blocks` 有普通组和超级组记录。
- `workout_sets` 有每一组动作明细。
- 如果动作已收藏，动作选择器顶部出现“常用收藏”，收藏动作也会在搜索结果中置顶。
- 推荐动作会随想练部位、可用器材、策略和身体状态变化；点击“加入”会创建普通训练块。
- 保存后最近训练中能看到刚保存的训练。

## 5. 训练详情验收

1. 在“训练”页点击最近训练。
2. 确认进入训练详情。
3. 确认标题、日期、场所、目标显示正确。
4. 确认普通组、超级组分块展示。
5. 确认每组的动作名、轮次、重量、次数正确。
6. 修改训练信息中的标题、日期、场所、目标和备注，点击“保存训练信息”。
7. 退出再进入详情，确认训练信息修改被保留。
8. 修改某一组的重量、次数或 RPE，点击“保存”。
9. 返回“档案”页确认统计已重算，最大重量或容量随修改更新。
10. 回到训练详情，删除一组误录数据。
11. 删除一个测试训练块，确认页面刷新后该块消失。

通过标准：编辑和删除操作只影响当前用户自己的训练记录；保存/删除后会调用 `recalculateStats`，档案页不再保留被删除训练组造成的旧统计。

## 5.1 复制上次训练验收

1. 在“训练”页找到“最近训练”。
2. 先切换“同目标”“同场所”“同目标+场所”筛选，确认列表数量和展示内容会随当前目标/场所变化。
3. 点击某条历史训练右侧“复制”。
4. 确认训练块、动作、重量、次数、RPE 被带入当前训练编辑区。
5. 修改标题、日期或某一组重量。
6. 点击“保存并结束训练”。
7. 打开最近训练中的新记录，确认它是新的训练记录，旧记录没有被修改。

## 6. 个人档案验收

1. 打开“档案”。
2. 点击“重算”。
3. 确认训练次数、有效组数、总次数、总容量更新。
4. 确认“近期训练摘要”显示近 7 天和近 30 天的训练次数、有效组数、总次数与容量。
5. 确认“PR 关注”最多展示 3 个带重量动作，并按估算 1RM（无 1RM 时按最大重量）从高到低排列。
6. 确认动作表现里显示最大重量和估算 1RM。
7. 点击“导出 JSON 备份”，确认生成 `GymGymGym-backup-日期.json` 并出现微信文件分享面板。
8. 打开导出的 JSON，确认包含 `workout_sessions`、`workout_blocks`、`workout_sets`、`exercise_stats` 和 `user_goals`。

通过标准：

- `recalculateStats` 云函数执行成功。
- `exercise_stats` 按当前 OpenID 写入统计。
- 如果云函数暂不可用，页面仍可基于最近 `workout_sets` 做轻量 fallback 展示。
- 近期摘要只统计对应时间窗口内的正式组，热身组不计入有效组、次数和容量。
- JSON 备份仅包含当前 OpenID 数据，导出过程不修改云端记录。

## 7. 真机验收

1. 微信开发者工具生成预览码。
2. 两个人分别扫码体验。
3. 两个人各保存一次训练。
4. 确认两个人只能看到自己的训练和档案。
5. 尝试用另一人的训练详情 URL/ID 进入，确认无法加载或修改该记录。
6. 确认小程序端不能直接写入 `exercise_stats`，档案统计只能通过 `recalculateStats` 更新。

## 8. 回归检查命令

每次修改后至少运行：

```bash
npm run check
npm test
git diff --check
```
