# GymGymGym 产品路线图

## 当前版本：MVP 私人可用版

目标是先让两个人可以稳定记录训练，而不是做成通用健身 App。

已覆盖：

- 今日训练场景记录：日期、场所、目标、状态、可用器械、备注。
- 普通组与超级组：一个训练块可包含 1 个普通动作，或 2–4 个超级组动作。
- 每组数据：重量、次数、RPE、热身、力竭。
- 动作库：基于 `hasaneyldrm/exercises-dataset` 的标准化动作数据，支持搜索与筛选。
- 个人档案：按动作汇总总组数、总次数、总容量、最大重量、估算 1RM。
- 近期分析：档案页展示近 7 天 / 近 30 天训练摘要和 PR 关注 Top 3。
- 目标设置：昵称、训练水平、默认目标、默认场所、常用器材、长期目标描述。
- 具体目标：支持创建多个重量、1RM、训练频率、目标体重或自定义指标目标，并管理完成状态。

## V0.2：更好用的训练记录

当前已完成：

- 最近训练一键复制，带入训练块、动作和上次重量/次数/RPE。
- 训练详情可编辑训练标题、日期、场所、目标和备注。
- 训练详情可编辑已保存组的重量、次数、RPE、热身/力竭。
- 训练详情可删除误录组或整个训练块，并自动重算档案统计。
- 统计重算会清理已经没有有效组的旧动作统计。
- 动作库支持收藏/取消收藏动作，训练动作选择器优先展示收藏动作。
- 训练页支持根据今天想练部位、场地器材、身体状态和突破/维稳策略推荐动作，并可一键加入训练块。
- 训练页最近训练支持按“同目标 / 同场所 / 同目标+场所”筛选复制来源。

仍可继续增强：

- 支持按训练部位、收藏模板或历史表现进一步推荐复制来源。

原始规划：

优先级高：

1. 编辑已保存训练
   - 修改某一组的重量、次数、RPE。
   - 删除误录的训练块或训练组。
   - 重算统计。
2. 复制上次训练
   - 从最近一次同类训练复制训练块和动作。
   - 默认带入上一轮重量与次数。
3. 常用动作（已完成基础版）
   - 收藏动作。
   - 动作选择器优先展示常用动作。
   - 后续可根据最近使用频次自动推荐收藏。

## V0.3：目标与建议

优先级中：

1. 目标拆解
   - 已完成结构化目标基础版：可记录“卧推 100kg”等具体目标及截止日期、指标和完成状态。
   - 后续增加阶段里程碑，并在档案页展示目标动作最近趋势与完成进度。
2. 场景化训练建议
   - 根据“今天想练什么 / 有什么器材 / 想突破还是维稳”推荐动作列表。
   - 基础轻量规则已完成；后续可加入历史表现趋势和疲劳间隔。
3. 训练提醒
   - 记录最近训练部位。
   - 给出“上次练腿距今 X 天”的提示。

## V0.4：数据分析

优先级低：

1. 周/月训练统计：已完成轻量版近 7 天 / 近 30 天摘要，后续可扩展图表。
2. 动作 PR 时间线：已完成 PR 关注 Top 3，后续可扩展按动作时间线。
3. 肌群训练频率热力图。
4. 导出备份：已完成 JSON 个人数据备份；后续可增加 CSV 和备份恢复导入。

## 暂不做

- 社交、排行榜、公开动态。
- 复杂课程市场或教练端。
- 体脂、饮食、睡眠等非核心模块。
- 多租户后台管理。

## 决策原则

- 私人使用优先：能减少手动输入的功能优先。
- 数据安全优先：训练数据只按 OpenID 隔离。
- 简单可维护优先：优先使用小程序端直连 CloudBase 文档数据库，只有统计重算和用户初始化使用云函数。
